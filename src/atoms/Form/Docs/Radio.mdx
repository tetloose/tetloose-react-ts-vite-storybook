# Radio

The **Radio** component is a flexible form control that renders a native `<input type={'radio'} />` element wrapped in a custom `<Label />` and a `Spacer`. It supports customisable styling through colour variants, modifiers, and error states.

## Usage

```tsx
import { Radio } from '@atoms/Form/_Radio.component'
```

```tsx
  <Radio
    padding={{ default: 2 }}
    color={color}
    htmlFor={'yellow'}
    id={'yellow'}
    value={'yellow'}
    label={{
      text: 'Yellow',
      size: 'body-med'
    }}
    error={!!colorsError}
    {...register('colors', { required: true })}
  />
  <Radio
    padding={{ default: 2 }}
    color={color}
    htmlFor={'blue'}
    id={'blue'}
    value={'blue'}
    label={{
      text: 'Blue',
      size: 'body-med'
    }}
    error={!!colorsError}
    {...register('colors', { required: true })}
  />
```

### modifiers

- Type: `string[]`
- Default: `[]`
- Description: Additional CSS classes for custom styling. These modifiers are appended to the component's base classes.

### color

- Type: `string`
- Default: `'dark'`
- Description: Determines the colour variant of the radio input, applied via the label's styling.

### label

- Type: `LabelProps['label']`
- Default: None
- Description: An object containing properties for the label. It is spread into the `<Label />` component to configure the label text and styling.

### padding

- Type: `Spacers`
- Default: None
- Description: Spacing applied to the component via the Spacer wrapper, controlling the internal padding.

### margin

- Type: `Spacers`
- Default: None
- Description: Margin applied around the component via the Spacer wrapper.

### error

- Type: `boolean`
- Default: `false`
- Description: Indicates whether the radio input is in an error state. When set to true, error styling is applied.

### ...rest

- Type: `InputHTMLAttributes<HTMLInputElement> (excluding the type property)`
- Default: None
- Description: Additional native HTML attributes for the `<input type={'radio'} />` element.
